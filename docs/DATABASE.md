# Database Design — AI Intelligence OS

## Overview

**Backend:** Supabase (managed PostgreSQL 15+)
**ORM / Query Layer:** Direct SQL via Supabase client (PostgREST) and n8n Postgres nodes
**Migration Strategy:** Sequential numbered SQL files in `database/migrations/`

### Why these tables?

| Table | Purpose |
|---|---|
| `sources` | Registry of every feed, API, or scraper endpoint that n8n polls. Central control plane for the ingestion pipeline. |
| `categories` | Controlled vocabulary for classifying news items. Drives UI grouping and daily report structure. |
| `news` | Immutable append-only store of every collected item. AI enrichment fields (summary, score, tags) are filled in after initial insert. |
| `daily_reports` | One aggregated digest per calendar day. Generated by AI from that day's processed news items. The deliverable the user actually reads. |

## Table Definitions

### `sources`

Registry of all content ingestion endpoints. Each row represents one feed that n8n polls on a schedule.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | `uuid` | NOT NULL | `gen_random_uuid()` | Primary key |
| `name` | `text` | NOT NULL | — | Human-readable display name, e.g. `"Hacker News"` |
| `slug` | `text` | NOT NULL | — | URL-safe machine key, e.g. `"hacker-news"`. Used in workflow IDs and logs. **UNIQUE** |
| `type` | `text` | NOT NULL | — | Ingestion method: `'rss'` \| `'api'` \| `'scraper'` \| `'youtube'` |
| `url` | `text` | NOT NULL | — | Base URL, RSS feed URL, or API endpoint |
| `is_active` | `boolean` | NOT NULL | `true` | Soft-disable a source without deleting its history |
| `check_interval_minutes` | `integer` | NOT NULL | `60` | How often n8n should poll this source. Lower = more frequent = higher API cost |
| `last_checked_at` | `timestamptz` | NULL | — | Timestamp of last successful poll. NULL means never checked |
| `error_count` | `integer` | NOT NULL | `0` | Consecutive error counter. n8n increments on failure, resets on success. Used for alerting |
| `metadata` | `jsonb` | NULL | — | Per-source configuration bag: API keys references, subreddit names, YouTube channel IDs, RSS namespaces, etc. |
| `created_at` | `timestamptz` | NOT NULL | `now()` | Row creation time |
| `updated_at` | `timestamptz` | NOT NULL | `now()` | Last modification time. Maintained by trigger |

**Constraints:**
- `sources_slug_key` — UNIQUE on `slug`
- `sources_type_check` — CHECK `type IN ('rss', 'api', 'scraper', 'youtube')`

---

### `categories`

Controlled vocabulary for classifying news. AI assigns one category per news item after collection. Categories are also used to structure the sections of a daily report.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | `uuid` | NOT NULL | `gen_random_uuid()` | Primary key |
| `name` | `text` | NOT NULL | — | Display name, e.g. `"Research Papers"`. **UNIQUE** |
| `slug` | `text` | NOT NULL | — | Machine key used as JSON keys in `daily_reports.content_by_category`. **UNIQUE** |
| `description` | `text` | NULL | — | Longer explanation of what belongs in this category. Fed to AI classifier as a hint |
| `color` | `text` | NULL | — | Hex color string (e.g. `"#8B5CF6"`) for UI badge rendering |
| `icon` | `text` | NULL | — | Icon name (Lucide icon set) for UI rendering, e.g. `"cpu"` |
| `is_active` | `boolean` | NOT NULL | `true` | Inactive categories are hidden from UI and excluded from report generation |
| `sort_order` | `integer` | NOT NULL | `0` | Integer sort position for consistent UI ordering |
| `created_at` | `timestamptz` | NOT NULL | `now()` | Row creation time |

**Constraints:**
- `categories_name_key` — UNIQUE on `name`
- `categories_slug_key` — UNIQUE on `slug`

---

### `news`

The central fact table. Every collected item lands here. AI enrichment fields (`content_summary`, `importance_score`, `tags`, `category_id`) start NULL and are populated by the processing pipeline after the initial insert.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | `uuid` | NOT NULL | `gen_random_uuid()` | Primary key |
| `source_id` | `uuid` | NOT NULL | — | FK → `sources.id`. Which source produced this item |
| `category_id` | `uuid` | NULL | — | FK → `categories.id`. NULL until AI categorization runs |
| `title` | `text` | NOT NULL | — | Article or item title, as-collected |
| `url` | `text` | NOT NULL | — | Canonical URL of the item |
| `content_raw` | `text` | NULL | — | Raw text from RSS `<description>`, scraped body, or API content field. May be HTML |
| `content_summary` | `text` | NULL | — | AI-generated 2–3 sentence summary. NULL until processed |
| `author` | `text` | NULL | — | Author name or handle, if available from source |
| `published_at` | `timestamptz` | NULL | — | Original publication timestamp from source. NULL if source doesn't provide it |
| `collected_at` | `timestamptz` | NOT NULL | `now()` | When n8n inserted this row. Used as fallback sort key when `published_at` is NULL |
| `importance_score` | `numeric(3,2)` | NULL | — | AI relevance score between `0.00` and `1.00`. NULL until processed |
| `tags` | `text[]` | NOT NULL | `'{}'` | AI-extracted keyword tags, e.g. `{llm, fine-tuning, open-source}` |
| `is_processed` | `boolean` | NOT NULL | `false` | True once the AI pipeline has run summarization, scoring, and categorization |
| `is_featured` | `boolean` | NOT NULL | `false` | Top-story flag. Set by AI when importance_score ≥ threshold, or manually by operator |
| `is_duplicate` | `boolean` | NOT NULL | `false` | True if this item is a duplicate of another |
| `duplicate_of` | `uuid` | NULL | — | Self-referential FK → `news.id`. Points to the canonical (first-seen) item |
| `url_hash` | `text` | NOT NULL | — | SHA-256 of the normalized URL. Used for O(1) duplicate detection on insert. **UNIQUE** |
| `metadata` | `jsonb` | NULL | — | Source-specific extra data: HN upvotes/comments, GitHub stars, Reddit score, etc. |
| `created_at` | `timestamptz` | NOT NULL | `now()` | Row insertion time |
| `updated_at` | `timestamptz` | NOT NULL | `now()` | Last modification time. Maintained by trigger |

**Constraints:**
- `news_url_hash_key` — UNIQUE on `url_hash`
- `news_importance_score_check` — CHECK `importance_score IS NULL OR (importance_score >= 0 AND importance_score <= 1)`
- `news_source_id_fkey` — FK → `sources.id` ON DELETE RESTRICT
- `news_category_id_fkey` — FK → `categories.id` ON DELETE SET NULL
- `news_duplicate_of_fkey` — FK → `news.id` ON DELETE SET NULL

---

### `daily_reports`

One row per calendar day. Aggregates that day's processed news into a structured digest. The JSONB columns are intentional denormalization — they snapshot news content at generation time so the report remains stable even if source rows are later modified.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | `uuid` | NOT NULL | `gen_random_uuid()` | Primary key |
| `report_date` | `date` | NOT NULL | — | The calendar date this report covers. **UNIQUE** — enforces one report per day |
| `title` | `text` | NOT NULL | — | Report headline, e.g. `"AI Intelligence Digest — July 6, 2026"` |
| `summary` | `text` | NULL | — | AI-generated executive summary paragraph for the day's news |
| `top_stories` | `jsonb` | NULL | — | Ordered array of curated top story objects: `[{id, title, url, summary, source, category}]` |
| `content_by_category` | `jsonb` | NULL | — | Object keyed by category slug, each value an array of news item objects |
| `total_items` | `integer` | NOT NULL | `0` | Count of all news items collected on this date |
| `processed_items` | `integer` | NOT NULL | `0` | Count of items that completed AI processing |
| `status` | `text` | NOT NULL | `'draft'` | Pipeline state: `'draft'` → `'generating'` → `'ready'` \| `'failed'` |
| `generated_at` | `timestamptz` | NULL | — | Timestamp when AI generation completed successfully |
| `generation_duration_ms` | `integer` | NULL | — | Wall-clock time for the AI generation step. Used for performance monitoring |
| `ai_model_used` | `text` | NULL | — | Model identifier, e.g. `"claude-sonnet-4-6"`. For reproducibility and cost attribution |
| `ai_tokens_used` | `integer` | NULL | — | Total input + output tokens consumed generating this report. For cost tracking |
| `metadata` | `jsonb` | NULL | — | Extensible bag for future fields (email send status, Slack post URL, etc.) |
| `created_at` | `timestamptz` | NOT NULL | `now()` | Row creation time |
| `updated_at` | `timestamptz` | NOT NULL | `now()` | Last modification time. Maintained by trigger |

**Constraints:**
- `daily_reports_report_date_key` — UNIQUE on `report_date`
- `daily_reports_status_check` — CHECK `status IN ('draft', 'generating', 'ready', 'failed')`

## Indexes

| Index Name | Table | Columns | Type | Rationale |
|---|---|---|---|---|
| `idx_sources_is_active` | `sources` | `is_active` | B-tree | n8n queries `WHERE is_active = true` to build its polling queue |
| `idx_sources_type` | `sources` | `type` | B-tree | Filter sources by ingestion method when debugging or routing |
| `idx_news_url_hash` | `news` | `url_hash` | B-tree (UNIQUE) | O(1) duplicate detection — every insert checks this before writing |
| `idx_news_source_id` | `news` | `source_id` | B-tree | Join and filter by source; also used in source-level stats queries |
| `idx_news_category_id` | `news` | `category_id` | B-tree | Filter uncategorized items (`IS NULL`) and group by category for reports |
| `idx_news_published_at` | `news` | `published_at DESC` | B-tree | Time-ordered fetches for report generation and the "latest" feed view |
| `idx_news_collected_at` | `news` | `collected_at DESC` | B-tree | Pipeline queries for items collected in a time window (e.g. last 24 hours) |
| `idx_news_is_processed` | `news` | `is_processed` | B-tree | AI processing queue: `WHERE is_processed = false ORDER BY collected_at` |
| `idx_news_is_featured` | `news` | `is_featured` | B-tree | Quick top-stories filter for daily report generation |
| `idx_news_importance_score` | `news` | `importance_score DESC NULLS LAST` | B-tree | Ranking queries for selecting top N items by score |
| `idx_news_collected_at_processed` | `news` | `collected_at, is_processed` | B-tree (composite) | Efficient processing queue scan with time bounds |
| `idx_news_tags` | `news` | `tags` | GIN | `WHERE tags @> ARRAY['llm']` — array containment search for tag filtering |
| `idx_daily_reports_report_date` | `daily_reports` | `report_date DESC` | B-tree (UNIQUE) | Primary access pattern: fetch report by date |
| `idx_daily_reports_status` | `daily_reports` | `status` | B-tree | Find in-progress or failed reports for monitoring |

## Foreign Key Relationships

```
news.source_id        → sources.id        ON DELETE RESTRICT
news.category_id      → categories.id     ON DELETE SET NULL
news.duplicate_of     → news.id           ON DELETE SET NULL   (self-referential)
```

**Design rationale:**

- `ON DELETE RESTRICT` for `source_id` — you must reassign or delete all news items before removing a source. Prevents orphaned records and forces intentional cleanup.
- `ON DELETE SET NULL` for `category_id` — deleting a category downgrades items to uncategorized rather than cascading a potentially large delete.
- `ON DELETE SET NULL` for `duplicate_of` — if the canonical item is deleted, duplicates become standalone items rather than pointing to a ghost.

## Naming Conventions

| Convention | Rule | Example |
|---|---|---|
| Table names | `snake_case`, plural | `daily_reports`, `news` |
| Column names | `snake_case` | `is_processed`, `url_hash` |
| Primary keys | Always `id uuid` | `id uuid DEFAULT gen_random_uuid()` |
| Foreign keys | `{referenced_table_singular}_id` | `source_id`, `category_id` |
| Boolean columns | `is_` prefix | `is_active`, `is_processed`, `is_featured` |
| Timestamp columns | `_at` suffix | `created_at`, `collected_at`, `generated_at` |
| Duration columns | `_ms` suffix for milliseconds | `generation_duration_ms` |
| Count columns | `_count` or `total_`/`processed_` prefix | `error_count`, `total_items` |
| Index names | `idx_{table}_{column(s)}` | `idx_news_url_hash` |
| Constraint names | `{table}_{column}_{type}` | `news_url_hash_key`, `sources_type_check` |
| Enum-like text fields | Document valid values in a CHECK constraint | `status IN ('draft', 'generating', 'ready', 'failed')` |

## Future Tables (Post-MVP)

These tables are designed but excluded from the initial migration to keep scope tight.

### `user_preferences`
Per-user configuration: preferred categories, importance threshold for notifications, report delivery time, timezone. Enables personalization when the app moves beyond single-user.

### `report_subscriptions`
Maps users to delivery channels (email, Slack, webhook) for daily reports. Currently handled by a single hardcoded n8n workflow.

### `source_health_logs`
Time-series log of each poll attempt: success/failure, latency, item count, HTTP status. Enables reliability dashboards and alerting thresholds beyond the simple `error_count` counter.

### `ai_usage_logs`
Per-call log of AI API usage: model, tokens in/out, latency, cost estimate, purpose (summarization vs. categorization vs. report generation). Enables cost attribution and budget tracking.
